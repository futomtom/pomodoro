PomodoroTableViewController

The PomodoroTableViewController works with the PomodoroModel to display the current list of pomodoro instances the user has created. Users can add new a new pomodoro using the + button in the navigation bar. (Edit was a feature I didn't get the chance to implement, unfortunately!) When the application first loads, the PomodoroTableViewController initializes the PomodoroModel (which will live for the duration of the app's life). The PomodoroModel has an initializer that loads all the user's pomodori from disk, using Core Data. Those pomodori are stored as NSManagedObjects in the model (in array pomodori), and then passed to the views that use them as Pomodoro structs. This is for convenience and to avoid having Core Data objects mucking around in the ViewControllers. Here is the structure of the Pomodoro struct:

struct Pomodoro {
var name: String
var intervalLength: Int
var numIntervals: Int
var breakLength: Int
}

AddNewViewController

The user can add a new pomodoro in this view. They must specify all parameters before PomodoroModel will store the values and send the notification that the change was made. Saving the pomodoro not only appends it to the pomodori array in memory, but also saves it to disk using Core Data.

PomodoroViewController

PomodoroViewController displays the selected indexPath.row from the PomodoroTableViewController and primarily interacts with TimerModel. (It only receives a copy of the indexPath.row Pomodoro struct, because it doesn't need to do anything with the PomodoroModel.) It imports AudioToolBox to vibrate the user device at every interval and break end. (The user can specify on the Settings view whether to disable this function of the app.)

SettingsViewController

This is an ultra vanilla implementation of settings options. I've not implemented persistence on user settings, but I would like to try using the pList to store some simple settings. I'd also like to implement the settings bundle. Finally, I'd like to embed the settings options into a UITableView, similar to the iOS Settings app.

In the near future, I'd like to implement functionality to let the user select whether to play a sound at the end of each interval/break.
